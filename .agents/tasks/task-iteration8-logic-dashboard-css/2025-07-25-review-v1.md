# Investment logic guards, cache controls, and dashboard polish

This iteration fixes several correctness issues in the investment analysis pipeline (Kelly criterion edge case, Lynch PEG overflow, Graham PE=0 penalty, overheating check using unavailable metadata), adds cache management APIs to the dashboard, and applies CSS/UX polish across multiple template pages. The dead-code `add_debate_message` body that sat unreachably after a `return` is now a proper method. Twenty-four new tests bring coverage of these edge cases to the suite.

**Watch for:** The `/api/cache/clear` endpoint uses GET with no auth, making it trivially callable by any browser or crawler—this is a state-mutating action exposed without protection (confirmed). The Kelly 1% minimum floor applies unconditionally to all bullish signals regardless of score or confidence, which subtly changes existing behavior for all positions (confirmed). The `max-width: 200px` ellipsis rule is applied broadly to `.data-value` which may clip non-name content like long numbers or descriptions (likely).

## High-level view

The investment logic fixes each address a clearly-observable edge case (division-by-near-zero in Lynch PEG, zero-score for missing PE in Graham, dead metadata lookup in overheating, zero allocation for score=5 bullish). The Kelly minimum floor is the most consequential because it introduces a hard 1% position for every bullish signal including those with score at threshold and low confidence—removing the ability to express "barely bullish, sit out."

The cache management additions (TTL reduction, `force_refresh`, `clear_cache()`, and two new API endpoints) are straightforward module-level dict operations. The main gap is `/api/cache/clear` being a GET request that mutates server state with no authorization—any client can flush the cache at will.

The CSS and template changes are conservative. The broad `.data-value` selector for the ellipsis rule could clip content unintentionally in non-name contexts.

## Dead code extraction in `add_debate_message`

On main, lines 458–459 of `registry.py` contained a docstring and `self._debate_history.append(msg)` after a `return result` statement—the string is a no-op expression, the append is unreachable. The `DebateProtocol` class elsewhere calls `coordinator.add_debate_message(...)`, which would raise `AttributeError` at runtime since the method didn't exist on the class. This was a genuine bug, not just dead code. The fix correctly extracts it into a proper method.

## Kelly criterion 1% minimum floor

```python
if pct < 1.0:
    pct = 1.0
```

This fires whenever score ∈ [5, ~5.1) because `edge = (5.0 - 5) / 5 = 0`, making `pct = 0`. Before this fix, a bullish consensus at exactly score 5.0 would result in 0% allocation despite the system labeling the signal as bullish. The floor applies unconditionally regardless of confidence—a signal with score=5.0 and confidence=0.3 now gets 1%. Whether this is desirable depends on how downstream portfolio managers interpret "1% position."

## Overheating check using actual context

The old code iterated `valid_results.values()` looking for `r.metadata.get("context")`, but no persona actually stores a `context` object in response metadata. The PE>30 warning never fired—it was dead logic. The fix uses `ctx_for_risk` (already constructed earlier in `get_consensus`), making the check functional for the first time.

## Cache management API: no authorization on state mutation

`/api/cache/clear` is a GET endpoint that calls `_cache.clear()`. In a single-user local deployment this is acceptable. In any shared or publicly-exposed deployment, this is a denial-of-service vector: an attacker can force rate-limit exhaustion against yfinance by alternating cache-clear and fetch requests. Using GET for a mutating operation also means browser prefetch, crawlers, and link-preview bots can inadvertently clear the cache.

## CSS ellipsis on `.data-value`

The new rule applies `overflow: hidden; text-overflow: ellipsis; white-space: nowrap; max-width: 200px` to `.data-value` elements. This class is used across dashboard cards for various content types—ticker symbols, agent names, but also potentially numeric values or longer descriptive text. The hard `200px` cap may truncate meaningful content in wider viewports. A safer approach: scope to `.persona-row .data-value` or use `max-width: 100%` with a parent constraint.

## sessionStorage persistence gap

When `runAllAnalysis` completes, results are saved to `sessionStorage['augur-analysis-results']`. When a ticker is deleted from the watchlist, the sessionStorage copy is not updated—the removed ticker's result lingers. If the user re-adds the same ticker before the session ends, stale results flash briefly before fresh analysis runs.
