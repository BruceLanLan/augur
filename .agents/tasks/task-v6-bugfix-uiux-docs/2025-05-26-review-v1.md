# v6.0 Release: Bug Fixes, UI/UX Overhaul, Documentation Restructure

Augur v6.0 is a release-polish pass touching 38 files across three concerns: fixing a deprecated FastAPI `TemplateResponse` call pattern and updating all persona-count references from 17 to 18, simplifying the dashboard interaction model so users get inline analysis results without page-hopping, and rewriting both READMEs into a tighter format with a dedicated changelog and English translation of integration guides. The version string moves from 5.5/5.6 to 6.0.0 across pyproject.toml, `__init__.py`, app.py, SVG metadata, and test assertions.

**Watch for:** Hard-coded "17" in the index page stats card that was not bumped to 18 (confirmed). CORS middleware omits `PUT` from `allow_methods`, breaking cross-origin config writes from Hermes (confirmed). Dead `autoFetch()` function left in stocks.html after the button was removed (confirmed).

---

## High-level view

The UI simplification removes the separate "auto-fetch" button from the stocks page—analysis now calls the API with `auto_fetch=true` when no manual metrics are provided, and the index page performs inline analysis directly rather than redirecting to `/stocks`. Preset buttons on the settings page let users batch-configure model assignments for all 18 personas in one click.

The documentation rewrite condenses the README by roughly 40%, replacing feature-matrix tables with prose and splitting the persona table into four thematic sub-tables. A new `docs/CHANGELOG.md` consolidates version history. Three new English-language integration guides were added.

The `scanner/` fallback import in `app.py` (line 38) still references the old module path. Both READMEs removed `scanner/` from the directory tree, creating an inconsistency: either the shim should be documented or the fallback import should be removed.

## Index page inline analysis

The index page previously redirected to `/stocks?ticker=X`. Now `heroGo()` fires `fetch('/api/analyze/' + t)` directly and renders a compact result card (signal badge, score, confidence, reasoning) inline below the hero input. Results are cached client-side via `augurCache` and rendered in a "recent analyses" list. The loading state uses skeleton shimmers and errors surface inline.

## Stocks page flow simplification and dead code

The new flow merges auto-fetch and analysis into a single action: `runAnalysis()` checks `hasUserMetrics()` and passes `auto_fetch=true` when no metrics were manually entered. The server-side handler already supported this parameter, so no API change was needed.

The `autoFetch()` function (~40 lines) remains defined in `stocks.html` but is now unreachable—no button or event handler references it. It should be removed.

## CORS `allow_methods` gap

```python
allow_methods=["GET", "POST"],
```

The config API exposes `PUT` (`/api/config`, `/api/config/persona/{id}`) and `DELETE` (`/api/watchlist/{ticker}`). A cross-origin frontend (Hermes sidebar) calling these will receive 405 Method Not Allowed. The fix is to add `"PUT"` and `"DELETE"` to the list, or use `["*"]`.

## Hard-coded "17" in stats

```python
{"value": "17", "label": "维度权重系统"},
```

Every other "17" reference in the codebase was bumped to 18. This one was missed, likely because it refers to a "dimension weighting system" count rather than persona count—but the number still appears stale in a release that explicitly brands itself as "18 masters".

## Preset button string matching

`applyPreset()` uses `options[i].value.toLowerCase().indexOf('claude')` to find matching models. If the available model list changes naming conventions (possible as providers version their APIs), the preset buttons will silently fail to change any selection. No toast or visual indicator tells the user that a preset couldn't find a matching model for a given persona.
