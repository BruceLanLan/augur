# Test coverage for debate/watchlist/IC, import cleanup, and type hints

This change adds 25 tests covering debate orchestration, watchlist/cron persistence, and IC backtest correctness, removes dead imports from 29 files across persona and bot modules, adds `logging.getLogger(__name__)` to modules that previously lacked it, and tightens return type annotations on public APIs in data, backtest, cron, and registry modules. The test suite grows from 275 to 300 passing tests. The import cleanup is mechanical and safe; the type-hint additions are non-behavioral string annotations that don't change runtime semantics.

Watch for: The `load_watchlist()` shallow-copy bug (documented in FEAT-001 findings but not fixed in this PR) â€” `DEFAULT_CONFIG.copy()` is a shallow copy, so the nested dicts inside it are shared (confirmed). The tests work around it by writing a clean YAML file before each case, but production code calling `add_to_watchlist` without an existing file on disk will mutate the module-level `DEFAULT_CONFIG` dict across calls.

## High-level view

The test file exercises real coordinator logic (not pure mock shells) for multi-round debate, watchlist CRUD, and Spearman IC calculations. Tests are well-isolated using tempdir fixtures and patched paths, though the MCP tool test reconstructs the tool's output format manually rather than calling the registered tool function directly â€” meaning format drift won't be caught.

The import cleanup is purely subtractive across 29 files. No functional change. The type hint additions use forward-reference strings (`"BacktestResult"`, `"BacktestRecord"`) where needed to avoid class-body scope issues in `backtest.py`.

## Debate test coverage and the mock boundary

The minority-report test correctly mocks `analyze_with_all` at the right seam (returning controlled signal distributions) while still verifying that `run_debate`'s post-processing logic appends `minority_report` to metadata. The debate-rounds test validates that `rounds=1` produces zero `[Debate` findings (since `rounds - 1 = 0` re-analysis iterations) and `rounds=3` produces non-zero â€” this locks the loop boundary.

The `test_augur_debate_mcp_tool` test is the weak link. It instantiates its own `AgentRegistry` and `DecisionCoordinator`, calls `run_debate` + `get_consensus`, then manually formats the output string using the same `f"  {result.agent_name:20s} | ..."` pattern. It imports `create_server` but never uses it. If the actual MCP tool's formatting diverges (spacing, header wording, field order), this test will still pass â€” it's testing its own format string, not the tool's.

## Watchlist tests and the DEFAULT_CONFIG shallow-copy issue

Every watchlist test writes a clean YAML to a tempdir before patching `WATCHLIST_PATH`, correctly avoiding the shallow-copy path. The underlying bug remains in production: when `~/.augur/watchlist.yaml` doesn't exist, `load_watchlist()` returns `DEFAULT_CONFIG.copy()`. Because `copy()` is shallow, the returned dict's `"notifications"` and `"schedule"` values are the same objects as those in `DEFAULT_CONFIG`. Code that then mutates these nested dicts (e.g., `config["notifications"]["telegram"]["enabled"] = True`) permanently corrupts the module-level constant for the process lifetime. The `_deep_merge` function already exists in the module and could serve as the fix (`_deep_merge(DEFAULT_CONFIG, {})` produces a fully independent copy).

## IC report tests: missing value-level assertions

`test_run_backtest_with_sample_data` redirects `RECORDS_DIR` and `RECORDS_FILE` to a tempdir, avoiding filesystem pollution. It asserts the presence of records and ICs but doesn't verify correctness of any specific IC value. A regression in the correlation formula could go undetected as long as it produces non-empty output. The `test_rank_correlation_*` tests partially compensate (they test the formula directly with known inputs), but there's no integration-level check that `run_backtest` feeds the right data into `_rank_correlation`.
